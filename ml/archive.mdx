# AI 学习资料归档说明

`ml/` 目录保留了大量历史学习资料、外部参考整理和实验性文稿。这些内容对研究和回溯有价值，但并不是每一份原始文件都适合继续作为 Mintlify 站点页面直接发布。

## 先判断你属于哪种需求

- 如果你只是想高效阅读，优先看 `/ml/index` 和 `/ml/collect`
- 如果你是为了翻历史资料、找原始来源或继续二次整理，再去看仓库里的 `ml/` 目录
- 如果你准备把某份原始资料重新做成站点页面，先补结构、说明、链接和 frontmatter，再决定是否纳入导航

## 当前发布策略

- 继续维护的站点入口页：
  - `/ml/index`
  - `/ml/collect`
  - `/ml/archive`
- 保留在仓库中的原始资料：
  - `ml/**/*.md`
  - 各主题目录下的历史草稿、原始翻译和参考文档

## 为什么这样处理

- 原始资料里的很多链接仍然沿用旧的相对路径写法，不符合当前站点规范。
- 这些文件更像“研究素材库”，不是已经完成清洗的产品文档。
- 继续把所有原始文件直接暴露在站点里，会让导航混乱，也会给 Mint 校验带来大量低价值噪音。

## 你仍然可以在哪里查看原始内容

- 仓库目录：`ml/`
- GitHub：[github.com/lilinji/GeneTind-docs/tree/main/ml](https://github.com/lilinji/GeneTind-docs/tree/main/ml)

## 后续整理原则

- 不删除原始资料文件。
- 只有在完成结构清洗、链接修复和说明补充后，才把某份原始资料重新纳入站点导航。
- 新增归档入口页时，优先用 MDX 做“目录页”，不要直接把未经清洗的 raw markdown 挂进导航。
